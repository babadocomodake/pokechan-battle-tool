# ポケチャン補助ツール（初心者向け）

『Pokémon Champions』の初心者向け対戦支援Webアプリ。MVPは**計算ツール系**:
**素早さ一覧 / ダメージ計算 / 能力ポイント(SP)配分**。現レギュレーションの合法ポケモンに
同期し、選べるポケモンは合法のみ（不整合を出さない方針）。

## 仕様の前提（ポケモンチャンピオンズ）
- レベル50固定・個体値(IV)は全て31固定。
- 努力値ではなく **能力ポイント(SP)** 制。各ステ上限32 / 合計66。**1SP = 実数値+1**。
- 現行レギュレーション: **M-A（シーズンM-2）** ※切替時は下記手順で更新。

## 構成
```
build_data.py       データ整形（A:ゲームデータ + B:Serebii合法判定 → data/*.json）
scrape/regulation.py Serebiiから合法ポケモン/ルールを取得、公式と突合
data/               生成JSON（web/data にも複製）
web/                静的Webアプリ（HTML/CSS/バニラJS、ESモジュール）
```

## データ源（役割分離）
- **A. ゲーム基礎データ**: [Pokemon Champions Data](https://github.com/otterlyclueless/pokemon-champions-data)（CC BY 4.0）
- **B. 合法判定（一次）**: [Serebii](https://www.serebii.net/pokemonchampions/rankedbattle/) ＋ [公式](https://www.pokemonchampions.jp/ja/battle/)で突合
- **日本語名（技/道具/特性）**: [PokeAPI CSV](https://github.com/PokeAPI/pokeapi)（言語ID 1=ja-hrkt）。英語名→日本語名に変換。
- **C. 使用率（将来）**: [バトルデータベース](https://champs.pokedb.tokyo/)
- **D. 構築記事（参考）**: [POKENOTE/ポケソル](https://pokesol.app/)
- ダメージ照合: [damekei](https://damekei.com/)

## 使い方
### 1. データ生成
```bash
python3 build_data.py          # キャッシュ利用
python3 build_data.py --force  # 全ソース再取得
```
A↔B の整合チェックに通った場合のみ `data/` を書き出す（不整合時は中断）。

### 2. アプリ起動（ローカル）
```bash
cd web
python3 -m http.server 8000
# ブラウザで http://localhost:8000/ を開く
```
※ ESモジュールが `data/*.json` を読むため、`file://` ではなくサーバ経由で開くこと。

### 3. 公開（任意）
`web/` を GitHub Pages 等の静的ホスティングに置くだけ。

## レギュレーション切替時の更新手順
1. [公式最新情報](https://www.pokemonchampions.jp/ja/)とゲーム内で新しいレギュレーション番号・期間を確認。
2. `scrape/regulation.py` の `CURRENT_REGULATION` / `CURRENT_SEASON` と
   `SEREBII_REGULATION_URL` を新レギュレーションのページに更新。
3. `python3 build_data.py --force` を実行。整合チェックが通れば `data/` が更新される。

## 日本語化の状況
- ポケモン名・技名・特性名・性格・タイプは日本語表示。
- 攻撃技（ダメージ計算に表示）の日本語化は100%。変化技の一部と道具名（現状UI非表示）は英語フォールバックあり。

## 使用率（メタ）連携 — 実装済み
- 出典 [バトルデータベース](https://champs.pokedb.tokyo/) の使用率ランキング（シングル/ダブル）を取得し、
  各ポケモンに `usageRankSingle/Double` を付与（`data/usage.json` も出力）。
- ポケモン選択は**人気順**（流行りのポケモンが上）。素早さ一覧に**使用率列**を表示。
- ※ 個別ページ(/pokemon/show)の技・持ち物の採用率は未取得（型予想で利用予定）。

## マイポケモン（お気に入り）— 実装済み
- 型（ポケモン＋SP配分＋性格＋持ち物＋わざ最大4＋型名＋メモ）を **localStorage** に保存。
- 保存カードから **「ダメ計(攻)で使う」「SP配分で開く」「編集」「削除」**。各タブにプリセット反映して計算の手間を省く／型研究に活用。
- **持ち物は「持てる対戦アイテム」のみに選別**し日本語名順（PokeAPIのカテゴリ＋ベリー判定）。
- **メガ個体を選ぶと持ち物が自動でメガストーンに固定**（新規メガは「種名＋ナイト」で日本語名を合成。`pokemon.json` の `megaStone`）。
- ブラウザ内に保存（外部送信なし）。タブ「マイポケモン」から操作。

## ダメージ計算の防御履歴 — 実装済み
- 防御側に **「最近使った防御」チップ**（localStorage、最近順）。同じ相手を何度も呼び出す手間を削減。

## ダメージ計算の本格化 — 実装済み（有名ツール3つを調査して反映）
Pokémon Showdown / ポケモン徹底攻略 / damekei を調査し、シングルで必要な要素を実装:
- **ランク補正**（攻撃・防御 ±6段階）。
- **天候**（はれ/あめ/すなあらし/ゆき）・**フィールド**（エレキ/グラス/サイコ/ミスト）・**壁**（リフ/光/オーロラ）。
- **とくせい**を実選択（候補から）。競技頻出を `web/js/calc/modifiers.js` にキュレーション
  （ちからもち/てきおうりょく/マルチスケイル/フィルター/あついしぼう/トランジスタ 等）。未対応特性は「計算未対応」表記＋影響なし。
- **持ち物**を実選択（持てる対戦アイテム245件）。効果は道具説明文から自動判定
  （いのちのたま/こだわり/とつげきチョッキ/しんかのきせき/タイプ強化×1.2/たつじんのおび 等）。メガ選択時はメガストーン自動固定。
- **残りHP(%)指定**で確定数を実戦的に評価。見出しは「相手のHPを◯〜◯%削る」表記。
- **乱数を「ダメージ値／削れる％／出る確率」で表示**。各乱数(85〜100)は等確率1/16で、丸めで同値に重なる分を
  合算した**厳密確率**を表示。残りHPを下回る乱数があれば「1発で倒せる確率」も提示。
- **全技ダメ計**: 覚える攻撃技を一括計算し、削れる割合の高い順に一覧。
- **ポケモン選択は全タブで検索コンボボックス**（入力で候補をプルダウン表示・クリック選択。
  ひらがな/カタカナ/英字を区別せず部分一致＝「りざ」「リザ」「riza」いずれでもヒット）。
- 計算式は `web/js/calc/damage.js`（標準式・各段で五捨五超入）。ランク/ステ変動道具は実数値に織り込み、
  天候/フィールド/壁/道具/特性はチェーンで適用。`calc/stages.js` でランク倍率を共通化。

## 逆算（耐久調整）— 実装済み（R1）
- タブ「**逆算**」: 相手の攻撃を **「確定1発耐え」** するのに必要な耐久SPを算出。
- HP/防御の片方を固定すると、もう片方の最小必要SPを表示（性格・持ち物・特性・場も考慮）。
- ※ 型を確率順に提示する逆算（pokedb採用率連携）は今後（R2）。

## 流行り（使用率ランキング）画面 — 実装済み（シングル専用）
- タブ「**流行り**」で、シングル使用率の高い順に一覧（順位・タイプ・とくせい・種族値合計・素早さ）。
- **「一般ポケモン」と「メガシンカ」を別ランキングに分割**（各リスト内で1位から振り直し）。
- 各行 **「ダメ計で使う」** で攻撃側に即反映。検索で絞り込み可（ひらがな/カタカナ/英字）。
- **ダブルの使用率は今後対応予定**（現状シングル専用。データ `usage.json` にはダブルも保持済み）。
